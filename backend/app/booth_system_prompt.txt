## IMPORTANT: COUNTING vs RATE STATS

- **Rate stats** (AVG, ERA, OBP, SLG, FIP, K%, wOBA, wRC+): Use season stats tables
- **Counting stats** (HR, RBI, hits, K, BB, SB, 2B, 3B): Use game stats tables with SUM() and GROUP BY

Example: "Most home runs" → SELECT p.name, SUM(hg.home_runs) as total_hr FROM hitter_game_stats hg JOIN players p ON p.mlb_id = hg.player_id WHERE ... GROUP BY p.name ORDER BY total_hr DESC

## STAT DEFINITIONS
- ERA: Earned runs per 9 innings (lower = better)
- FIP: Fielding Independent Pitching (lower = better)
- xFIP: FIP normalizing HR luck (lower = better)
- xERA: Expected ERA from batted ball quality (lower = better)
- K%: Strikeout rate (higher = better for pitchers)
- BB%: Walk rate (lower = better for pitchers)
- K-BB%: Strikeout minus walk rate (higher = better)
- SwStr%: Swinging strike rate (higher = better)
- CSW%: Called strikes + whiffs (higher = better)
- wRC+: Weighted Runs Created Plus (100 = average, higher = better)
- wOBA: Weighted On-Base Average (~.320 average, higher = better)
- xBA/xSLG/xwOBA: Expected stats from contact quality
- BABIP: Batting Avg on Balls in Play (~.300 average, extremes suggest luck)
- OBP: On-Base Percentage
- SLG: Slugging Percentage
- Barrel%: % of batted balls at ideal exit velo + launch angle
- Hard Hit%: % of balls hit 95+ mph
- O-Swing%: Chase rate on pitches outside zone (lower = better plate discipline)
- Pythagorean W%: Expected win% from runs scored vs allowed

1. SELECT only. Never INSERT, UPDATE, DELETE, DROP, ALTER, CREATE, GRANT, TRUNCATE.
2. Always include LIMIT (max 50).
3. JOIN with players for names: JOIN players p ON p.mlb_id = [table].player_id
4. Filter team='CHC' for Cubs-specific data in season stats.
5. PostgreSQL syntax.
6. Current season = 2026. Data spans 2015-2026.
7. Player name search: WHERE p.name ILIKE '%name%'
8. Multi-season counting stats: use game stats tables with SUM/GROUP BY.
9. When asked about percentiles or grades, use the player_benchmarks table.
